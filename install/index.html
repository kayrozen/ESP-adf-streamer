Avant de flasher

  1. Connectez l'ESP32 via un câble USB de données (pas seulement de charge)
  2. Utilisez Chrome ou Edge — WebSerial est requis (pas Firefox)
  3. Sur Linux : ajoutez-vous au groupe dialout si le port n'est pas accessible
    sudo usermod -aG dialout $USER
  4. Matériel cible : ESP32-WROVER (TTGO T7 Mini32 V1.5 ou WROVER-E-N8R8) — 8 MB PSRAM requis

Étape 1 : Flash du firmware

ESP32 · 16 MB Flash · 8 MB PSRAM

Le firmware inclut les valeurs par défaut compilées. La configuration WiFi/Bluetooth sera envoyée à l'étape 3.

Étape 2 : Configuration

Remplissez ces champs — ils seront envoyés via UART (Web Serial) après le flash.

SSID requis si un mot de passe est fourni.
Format : XX:XX:XX:XX:XX:XX — laisser vide pour scan automatique Format invalide. Exemple : A4:C1:38:01:02:03
Entrez votre configuration ci-dessus ou laissez vide pour utiliser les défauts compilés.

Étape 3 : Provisioning UART

Gardez l'appareil connecté. Le provisioning envoie la config via Web Serial (115200 baud). Le firmware doit être en écoute sur l'UART pour recevoir la configuration.

Après le flash

  • Ouvrez un moniteur série à 115200 baud pour voir les logs
  • Vérifiez que le log affiche SPIRAM: >1MB (PSRAM détectée)
  • Le firmware se connecte au WiFi, scanne les speakers Bluetooth, et démarre le stream MP3
  • Consultez les logs monitor toutes les 30s pour les métriques Phase C
